# System Workflow

## Repository Analysis Workflow

1. User enters GitHub repository URL.
2. Repository is validated.
3. Repository is cloned locally.
4. Files are scanned.
5. Statistics are generated.
6. Project structure is displayed.

---

## RAG Workflow

1. Repository files are scanned.
2. Files are divided into chunks.
3. Embeddings are generated.
4. Chunks are stored in vector storage.
5. User asks a question.
6. Relevant chunks are retrieved.
7. Context is sent to the LLM.
8. Answer is generated.

---

## AI Agent Workflow

1. User selects a file.
2. File content is loaded.
3. Agent-specific prompt is created.
4. Prompt is sent to the LLM.
5. AI response is generated.
6. Results are displayed.

---

## Export Workflow

1. User generates analysis.
2. User selects export format.
3. Report is created.
4. Report becomes available for download.
